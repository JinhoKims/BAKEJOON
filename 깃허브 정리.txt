- Git의 다운로드 구조 Clone, Pull의 차이 - 
Clone : 다운로드(설정폴더를 포함해서 전체소스를 다운 및 새폴더 생성)
Pull : 다운로드(git fetch + git merge) // 특정 파일만 로컬폴더에 다운로드해 갱신하는 개념
git Fetch : 로컬폴더에 서버에서 최신코드를 받아서 업데이트
git Merge : 두 개의 branch를 병합

- Git의 파일 업로드 구조 -

add "file1" : 올릴파일 선택
add "file2" : (올릴파일은 add로 여러번 선택해야 한다.)
그 후 
Commit "comment" : add한 파일들 업로드 준비("add한 파일들을 최종승인시킨다.") 
Push : 파일 올리기
